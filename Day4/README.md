1. Technical Writing One (Fundamentals)

Purpose :
Teach the fundamentals of technical writing — clarity, conciseness, and audience awareness.
Key Skills Learned :
Use terminology consistently.
Recognize ambiguous pronouns and fix them.
Distinguish active vs passive voice and convert sentences.
Develop strategies for clearer, shorter sentences.
Understand when to use bulleted vs numbered lists.
Create helpful lists and paragraph structure.
Define document scope and target audience.
Break long topics into sections.

2. Technical Writing Two (Intermediate Techniques)

Purpose :
Advance your writing beyond the basics, focusing on refinement and structure.
Core Topics :
Self-editing and revision strategies.
Organizing large documents effectively.
Introducing document scope and prerequisites.
Crafting visual communication and illustrations.
Creating useful sample code at varying complexity.
Identifying documentation types and writing tutorials.
Using LLMs for drafting & editing writing.

3. Writing Helpful Error Messages (Error Messages Course)

Purpose :
Teach how to write clear, actionable error messages that improve user experience.
Essential Principles :
Good error messages clearly answer:
What went wrong?
How to fix it?
Target error messages to the intended audience (engineer vs end-user).
Be concise but not cryptic; avoid vague or confusing language.
Identify the cause of the error and, where appropriate, the invalid user input.
Explain requirements and constraints clearly.
Use consistent terminology (don’t mix synonyms).
Carefully format and tone error messages; avoid double negatives.
